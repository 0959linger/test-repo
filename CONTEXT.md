# 项目定位与相关领域关系

## 这个项目不是什么

- **不是情感分析** — 不做分类，没有"正面/负面"标签
- **不是文本分类** — 不预设类别，没有训练数据
- **不是生成模型** — 不产出句子，不预测下一个 token
- **不是占卜工具** — 八卦是物理空间方位，不是语义映射
- **不是 LLM 插件** — 不需要大模型在线运行，完全独立
- **不是特征工程** — 铁律明确禁止人为特征提取

## 这个项目是什么

一个**物理格式的文本感知器**。

把文本当作物理信号注入八卦空间，让动力学系统（冲刷、级联、物极必反、冷却、凝结）自然演化，产出的卦象分布是物理浮现的结果，不是人为定义的分类。

## 与相关领域的关系

### 有远亲关系

| 领域 | 相似点 | 不同点 |
|------|--------|--------|
| Reservoir Computing / Echo State Networks | 物理动力学系统做计算，不改内部权重 | 通用时序处理，不用文本物理信号，没有八卦空间 |
| Hyperdimensional Computing | 高维向量做几何计算，分布式表示 | 目的是逻辑运算和记忆，不是物理浮现 |
| Ising 模型做文本分析 | 统计物理模型应用于文本 | 做主题建模/聚类，不是场演化 |
| Holographic Reduced Representations | 分布式高维表示 + 绑定/解绑操作 | 编码/解码范式，不是演化浮现 |

### 本项目的独特组合

以下每项单独看不算全新，但**组合在一起**在现有文献中没有先例：

1. **八卦作为物理空间** — 不是语义标签（乾 ≠ 天 ≠ 领导），是物理空间的 8 个方位
2. **汉字码点 + 位图密度作为物理信号** — 不是 token embedding 查表，是字符本身的物理属性
3. **物理浮现替代分类** — 不是 softmax 匹配最近类，是场自己决定哪条最稳
4. **七条铁律作为硬约束** — 不筛选、不特征提取、不投影、不匹配、不量化、不加权、不仲裁
5. **隔离演化** — 多条热源不做加权合并，在场内互推互拉，场自决收敛方向

### 与 LLM 的关系

这个引擎不是 LLM 的替代品，也不是 LLM 的插件。它是一个**平行的感知格式**：

| | LLM | 八卦物理引擎 |
|---|-----|-------------|
| 输入 | token 序列 | 物理信号 |
| 处理 | 注意力机制 | 物理演化（冲刷/级联/凝结） |
| 输出 | 下一个 token | 卦象分布 + 结晶词 |
| 可解释性 | 低（黑箱） | 高（每一步物理过程可追踪） |
| 确定性 | 低（采样随机） | 高（同输入 → 同输出） |

可以作为 LLM 的物理格式化前层，但引擎本身完全独立运行。

## 开发方式

本项目由人类与 AI 智能体协作开发完成。人类负责物理直觉、架构约束和铁律审查，AI 智能体负责代码实现、测试验证和架构审计。

这种开发方式本身也是引擎思想的一部分——人定义物理约束，机器在约束内探索，场自决结果。
