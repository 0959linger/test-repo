# v92 题实测报告 — 2026-06-12

## 方法
- 4个模型文件服务8卦位，每卦位独立推理
- 15道题（数学×3 + 代码×3 + 文学×2 + 哲学×1 + 常识×3 + 创意×2 + 抽象×1）
- 每道题8卦位各推理1次，记录logprob→factor

## 核心发现

### 1. 模型能力分化真实存在

| 题型 | 最强模型 | 说明 |
|:----|:--------:|:-----|
| 文学-诗词 | **3B** (1.016) | 大模型文学理解优势 |
| 抽象-比喻 | **3B** (1.001) | 大模型抽象能力优势 |
| 哲学-伦理学 | **3B** (1.011) | 大模型推理优势 |
| 代码-数据结构 | **C1.5** **(1.055)** | Coder模型真正优势领域！ |
| 常识-历史 | **C1.5** (1.091) | 知识密集型题 |
| 数学-概率 | **1.5B** (1.093) | 意外胜出 |
| 数学-微积分 | **C1.5** (1.136) | Coder模型数学强 |
| 代码-Python | **0.5B** (1.065) | 简单题小模型更自信 |

### 2. Coder模型验证成功
**代码-数据结构题**是C1.5唯一明显胜出的场景：
```
C1.5=1.055  vs  3B=0.909  →  Coder真正在代码结构题上更强
```

### 3. factor spread分布
- 范围: 0.0553 ~ 0.2089
- spread大(>0.15): 某些模型特别擅长/不擅长
- spread小(<0.06): 所有模型都能处理的简单题

### 4. 关于0.5B反常高分的解释
0.5B在某些题上factor超过3B（如SVD题=1.069 vs 3B=1.067）：
- **这不是"0.5B比3B能力强"**
- 而是小模型在简单问题上**概率分布更尖锐**（输出更确定），logprob更高
- 这是物理系统的自然行为：它不评判"谁更好"，只追踪"谁对当前信号反应更强"

## 结论
- v92架构的factor确实反映了不同模型在不同领域的**相对优势**
- 这种差异是**自然的、物理的、无仲裁的**
- **Coder模型在代码结构题上的优势被logprob正确捕获** ✅
- 系统不评判"哪个模型更好"，只记录"哪个模型对当前信号更确定"
