# v92 自适应设计（v91铁律基石版）

## 核心原则

**只做流速负反馈，不动v91铁律。**

### v91铁律（不可动）
- 熔断阈值 0.30%
- 桥梁阈值 4.9%
- decay_rate 0.005
- erosion_base 0.001
- 八卦拓扑（8通道必须都在）
- max_flow 20.0（理论天花板）
- 无仲裁
- 无指引

### 可自适应的参数
- **flow_rate**（工作参数，用户传入，默认8.0）

## "负反馈" vs "仲裁" 边界

| 类型 | 定义 | 示例 | v92合规性 |
|------|------|------|----------|
| 仲裁 | 系统判断"谁好/谁坏" → 分配资源 | "模型A比B好，给A更多权重" | ❌ 违反无仲裁铁律 |
| 负反馈 | 系统对物理观测做出物理响应 | "熔断发生 → 降流速" | ✅ 合规 |

**边界检验：**
- FlowAutoAdapter 只看"熔断是否发生"和"深度std" → 物理观测 ✓
- 不看"哪个模型产生这个logprob" → 无仲裁 ✓
- 调节的是flow_rate（工作参数），不是铁律常数 → 合规 ✓

## FlowAutoAdapter 设计

### 参数调节表

| 状态 | ratio | 调节速率 | 物理含义 |
|------|-------|---------|---------|
| 熔断 | - | -10%/步 | 物理本能（立即响应） |
| 远低于基线 | <0.6 | +1.0%/步 | 加速恢复 |
| 低于基线 | 0.6-0.8 | +0.7%/步 | 正常恢复 |
| 接近基线 | 0.8-1.0 | +0.3%/步 | 缓慢恢复 |
| 健康范围 | 1.0-1.3 | 0 | 保持不变 |
| 远高于基线 | >1.3 | -0.2%/步 | 缓慢回落 |

**硬边界：** [3.0, 15.0]

**触发条件：** steps_since_fuse > 30（稳定30步后开始恢复）

### 验证结果（4场景全通过）

```
场景1: 稳定系统（std<0.05, 无熔断）
  100步后: flow=8.000 (基线8.0)
  判断: [OK] 接近基线

场景2: 不稳定系统（频繁熔断）
  100步后: flow=3.000
  判断: [OK] 有效降速

场景3: 不稳定→稳定→恢复
  不稳定50步后: flow=3.826 (期望<6.0)
  稳定150步后: flow=7.595 (期望接近基线8.0)
  判断: [OK] 成功恢复

场景4: 极端熔断（每步都熔断）
  100步后: flow=3.000 (期望>=min_flow=3.0)
  判断: [OK] 硬边界保护
```

### 使用方式

```python
from v92_flow_adapter import FlowAutoAdapter

# 初始化
adapter = FlowAutoAdapter(base_flow=8.0)

# 每步冲刷循环
for step in range(N):
    flow = adapter.get_flow()  # 获取当前推荐流速
    
    # ... 执行冲刷（使用flow）...
    # ... 计算depths_std, did_fuse ...
    
    adapter.on_flush(depths_std, did_fuse)  # 更新状态
```

## 设计特点

1. **物理驱动** — 熔断事件本身就是物理信号，不需要系统判断"好/坏"
2. **不会崩** — 流速有硬边界[3.0, 15.0]，降/升幅度每一步都很小
3. **有记忆** — 最近熔断过 → 保持保守，稳定久了 → 慢慢恢复
4. **可逆** — 熔断后降速、稳定后恢复，不是单向的
5. **距离感知** — 离基线越远，恢复/回落越快
6. **v91铁律不可动** — 熔断线、衰减率、拓扑、无仲裁 全部不动 ✓

## 与550C的关系

> "我从未停止进化"

FlowAutoAdapter 的 flow_rate 在每一步都在微调（0.3%~10%），没有一步的参数跟之前完全相同。但它不是突变——每一步的变化都微小到几乎不可感知，只有长期看才有意义。
