# 八卦构架2 测试框架

## ⚠ 重要说明：兑/离/震/巽 零出现现象

在某些测试中，你可能会观察到兑、离、震、巽这4个卦位从未成为 winner（胜卦）。

**这是正常的场自决行为，不是bug。**

### 现象

- 零出现卦位的 distribution 值通常在 **0.10-0.13** 之间
- 所有8个卦位都有非零分布（**没有任何卦位为0**）
- 乾/坤/艮/坎 的峰值通常更高（0.15-0.25），更容易成为 winner

### 根因

1. **场选择最稳定的卦位**
   - v94 场演化会选择能量最集中、最稳定的卦位作为 winner
   - 乾/坤/艮/坎 的峰值更高，更稳定
   - 兑/离/震/巽 的峰值较低，很少成为最稳定的选择

2. **分布是连续的**
   - 所有卦位都参与计算，都有非零分布
   - 符合铁律：场自决，不人为干预

3. **测试输入的特性**
   - 当前测试用例偏向"稳定"场景
   - 缺少"变化"场景（冲突、转折、创新）

### 验证方法

详细分布分析见 `test_distribution_detail.py`：

```bash
python test_distribution_detail.py
```

输出示例：
```
输入: 今天天气很好
  胜卦: 艮
  分布（降序）:
    艮: 0.1613
    乾: 0.1416
    兑: 0.1241  ← 零出现卦位，但有非零分布
    坎: 0.1191
    坤: 0.1161
    离: 0.1138  ← 零出现卦位，但有非零分布
    震: 0.1127  ← 零出现卦位，但有非零分布
    巽: 0.1113  ← 零出现卦位，但有非零分布
```

### 如果想看到更多卦位变化

1. 增加测试场景多样性（冲突、创新、变化类问题）
2. 调整 v94 场演化参数（让场更敏感）
3. 调整汉字物理的权重（让物理线影响更大）

### 详细分析

见 `MEMORY.md`「兑/离/震/巽 零出现现象分析」章节。

## 测试框架概述

### 测试分类

- `test_dim1.py` - 汉字物理感知测试
- `test_dim2.py` - 河图语义感知测试
- `test_dim3.py` - qi_field 物理场测试
- `test_dim4.py` - 三线汇合测试
- `test_dim5.py` - 外围层测试（记忆/微风/C层/指涉层）
- `test_arch_integrity.py` - 架构完整性测试
- `test_engine_v9.py` - engine_v9 单元测试
- `test_gua_distribution.py` - 卦位分布分析
- `test_distribution_detail.py` - 详细分布分析（验证场自决行为）

### 运行测试

```bash
# 运行所有测试
pytest test_framework/

# 运行特定测试
pytest test_framework/test_dim1.py
pytest test_framework/test_arch_integrity.py

# 运行卦位分布分析
python test_framework/test_gua_distribution.py

# 运行详细分布分析
python test_framework/test_distribution_detail.py
```

### 性能指标

当前测试套件（69个非slow测试）：
- 有磁盘缓存：**0.32s**
- 无磁盘缓存：~5min（每个 EngineV9 实例 ~5s）

### 磁盘缓存

测试框架使用类似游戏着色器缓存的机制：

- `emb_256_normed_cache.npy` (148.5MB) - 归一化 256D 嵌入表
- `tokenizer_cache.pkl` - GGUF tokenizer 映射

缓存机制：
1. 首次运行：计算 + 存盘（~5s）
2. 后续运行：从磁盘加载（~0.3s）
3. 同进程内：内存缓存（~0.0s）

详见 `磁盘缓存说明.md`。
