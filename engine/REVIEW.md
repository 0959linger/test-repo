# 引擎全局审查 — 2026-07-07

## 一、引擎是什么（正确定位）

```
文本 + 时间 + 人物 + 历史 → qi 认知状态
```

**引擎 = 物理感知器。** 它不分类、不回答、不判断。它感知——把输入的物理结构映射到 qi 空间的一个位置。

## 四、走偏的地方

我们对引擎做的事 vs 引擎该做的事：

| 我们让它做的 | 它该做的 |
|------------|---------|
| 判断这道题是数学还是哲学 | 感知这道题的 qi 状态 |
| 追求 80% 分类准确率 | 诚实反映物理结构 |
| 和 NLP/GPT 比分类能力 | **不存在比较——根本不是同一类系统** |

**根因：把引擎当分类器用。引擎是感知器。**

## 五、正确的引擎定位

```
输入层（文本+上下文）
  ↓
物理感知层（码点+密度+时间+人物+历史 → qi）
  ↓
认知输出层（qi状态 → 文字）  ← 我们还没建
```

引擎的核心产出不是"类别标签"，而是**认知状态描述**：

- 当前 qi 分布：哪个方向强、哪个方向弱
- 确定度（CV）：场是收敛还是发散
- 圈数：停在8卦还是扩散到64卦
- 上下文漂移：和历史基线相比偏了多少
- 人物气场：对话中积累了什么倾向

**引擎的"回答"应该是这些状态的文字化描述——不是"这是数学题"，是"当前场在乾位高度凝聚，确定度0.91，人物气场偏坤，历史对话未造成明显偏移"。**
