TRUTH = """
一个32×32的汉字位图，从自编码器视角看，重构信息量≈3-4维。

# 三条路的诚实分析

THREE_PATHS = """
1. CNN做投影（不重建） + v94端到端

   → 问题是：没有了重建约束，CNN会学到什么？
   → 答：会学到一个对v94有利但对字形无意义的投影
   → 相当于手动编造8D，只是用梯度下降代替手工

   → 8个固定角度去掉，让盆地位置从数据长出来
   → 问题是：3-4维的数据，盆地最多长3-4个
   → 8个盆地是预定义的，3-4个盆地也是预定义的（只是数量不同）
   → "发现的不是定义的"这个原则在这里失效

3. 认了坍缩
   → CNN诚实输出3-4卦 = 字母层面的物理真相
   → 但我们已经有了引擎的输出（8/8卦，稳定，可解释）
   → CNN没有提供新维度，只是提供了更差的版本
"""
